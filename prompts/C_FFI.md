# TASK: Generate a pure-C header that exposes the Rust implementation across the ABI boundary using cbindgen.

## Execution Mode
**Autonomous action** — create the cbindgen config, generate the header, validate it compiles as both C and C++, then stop.

This prompt operates on a **Firefox source tree**. The target file-pair name is provided above.

## Context
Read the becrabbening documentation:
- [03-PHASE-2-C-FFI.md](./03-PHASE-2-C-FFI.md) — detailed Phase 2 instructions
- [examples/nsfoo/cbindgen.toml](./examples/nsfoo/cbindgen.toml) — worked example config
- [examples/nsfoo/nsfoo_ffi.h](./examples/nsfoo/nsfoo_ffi.h) — worked example header

## Workflow

### Step 1: Create `rust/{name}/cbindgen.toml`

```toml
language = "C"

header = "/* This file is generated by cbindgen. Do not edit by hand. */"
include_guard = "fox_{name}_ffi_h"
autogen_warning = "/* Warning: This file is autogenerated by cbindgen. Do not edit manually. */"

[export]
prefix = "Fox"
```

### Step 2: Run cbindgen

```bash
cbindgen --config rust/{name}/cbindgen.toml \
         --crate {name} \
         --output {name}_ffi.h
```

Run from the `rust/{name}/` directory, or adjust paths accordingly.

### Step 3: Audit the Generated Header

Hand-check `{name}_ffi.h`:
- All exported functions are present (compare against Phase 0 API snapshot).
- All opaque handle typedefs are present (`typedef struct FoxName FoxName;`).
- All enums are present with correct values.
- Include guard is set correctly.
- No C++ types have leaked into the header.

### Step 4: Verify Pure C

```bash
gcc -xc -fsyntax-only {name}_ffi.h
```

This must produce no errors and no warnings.

### Step 5: Verify C++ Compatibility

The shim will include this header inside `extern "C" {}`. Verify it works:

```bash
g++ -xc++ -fsyntax-only -c - <<EOF
extern "C" {
#include "{name}_ffi.h"
}
EOF
```

## Header Requirements

The generated header must satisfy ALL of the following:
- C89/C11 compatible — no C99 VLAs, no C++ features
- No classes, templates, namespaces, exceptions, references
- All functions prefixed `fox_{name}_*`
- Opaque handles as `typedef struct FoxName FoxName;`
- `#ifdef __cplusplus extern "C"` guards (cbindgen adds these automatically)
- Include guard: `#ifndef fox_{name}_ffi_h` / `#define fox_{name}_ffi_h` / `#endif`
- `#include <stdint.h>` and other required system headers

## Output Artifacts
- [ ] `rust/{name}/cbindgen.toml` created
- [ ] `{name}_ffi.h` generated and committed
- [ ] Header compiles cleanly with `gcc -xc -fsyntax-only`
- [ ] Header compiles cleanly in `extern "C" {}` block with `g++`

## What NOT to Do
- Do **not** hand-edit the generated header. Fix Rust source or `cbindgen.toml` and regenerate.
- Do **not** include C++ headers from `{name}_ffi.h`.
