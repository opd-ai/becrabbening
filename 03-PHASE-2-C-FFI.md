# Phase 2 — C FFI Interface

**Goal:** Generate a pure-C header that exposes the Rust implementation across the ABI boundary.

Previous: [02b-ANTI-SLOP-AUDIT.md](./02b-ANTI-SLOP-AUDIT.md) — Anti-slop audit (Rust quality gate).
Next: [04-PHASE-3-CPP-SHIM.md](./04-PHASE-3-CPP-SHIM.md) — C++ shim layer.

---

## Step-by-Step Instructions

### Step 1 — Create `rust/{name}/cbindgen.toml`

See [examples/nsfoo/cbindgen.toml](./examples/nsfoo/cbindgen.toml) for a complete example. Minimum configuration:

```toml
language = "C"

header = "/* This file is generated by cbindgen. Do not edit by hand. */"
include_guard = "fox_{name}_ffi_h"
autogen_warning = "/* Warning: This file is autogenerated by cbindgen. Do not edit manually. */"

[export]
prefix = "Fox"
```

See the [cbindgen.toml Reference](#cbindgentoml-reference) section below for all configuration fields.

### Step 2 — Run cbindgen

```bash
cbindgen --config rust/{name}/cbindgen.toml \
         --crate {name} \
         --output {name}_ffi.h
```

Run this from the `rust/{name}/` directory, or adjust paths accordingly.

### Step 3 — Audit the generated header

Hand-check the generated `{name}_ffi.h`:

- All exported functions are present (compare against Phase 0 API snapshot)
- All opaque handle typedefs are present (`typedef struct FoxName FoxName;`)
- All enums are present with correct values
- Include guard is set correctly
- No C++ types have leaked into the header

### Step 4 — Verify the header is pure C

```bash
gcc -xc -fsyntax-only {name}_ffi.h
```

This must produce no errors and no warnings.

### Step 5 — Verify the header also compiles as C++

The shim will include this header inside `extern "C" {}`, but verify it is syntactically valid C++ as well:

```bash
g++ -xc++ -fsyntax-only -c - <<EOF
extern "C" {
#include "{name}_ffi.h"
}
EOF
```

## `cbindgen.toml` Reference

| Field | Recommended Value | Description |
|---|---|---|
| `language` | `"C"` | Generate C, not C++ |
| `header` | `"/* generated */"` | Comment at top of file |
| `include_guard` | `"fox_{name}_ffi_h"` | C include guard macro name |
| `autogen_warning` | `"/* Do not edit */"` | Warning comment after header |
| `[export] prefix` | `"Fox"` | Prepend to all exported type names |
| `[export] include` | list of names | Whitelist specific items to export |
| `[export] exclude` | list of names | Blacklist specific items |
| `[parse] parse_deps` | `false` | Do not parse dependency crates |
| `no_includes` | `false` | Whether to suppress `#include <stdint.h>` etc. |

---

## Header Requirements

The generated header must satisfy all of the following:

- **C89/C11 compatible** — no C99 VLAs, no C++ features
- **No C++ features** — no classes, templates, namespaces, exceptions, references
- **All functions prefixed `fox_{name}_*`** — enforced by the `#[no_mangle]` names in Rust
- **Opaque handles** as `typedef struct FoxName FoxName;` — not a full struct definition
- **`#ifdef __cplusplus extern "C"` guards** — cbindgen adds these automatically when `language = "C"`
- **Include guard** — `#ifndef fox_{name}_ffi_h` / `#define fox_{name}_ffi_h` / `#endif`
- **`#include <stdint.h>`** and other required system headers — cbindgen adds these automatically

See [examples/nsfoo/nsfoo_ffi.h](./examples/nsfoo/nsfoo_ffi.h) for a complete example.

---

## What NOT to Do

- **Do not hand-edit the generated header.** If the output is wrong, fix the Rust source or `cbindgen.toml` and regenerate. A hand-edited header will drift from the Rust implementation.
- **Do not include C++ headers** from `{name}_ffi.h`. It must remain pure C.

## Output Artifacts

At the end of Phase 2, you should have:

- [ ] `rust/{name}/cbindgen.toml` created
- [ ] `{name}_ffi.h` generated and committed
- [ ] Header compiles cleanly with `gcc -xc -fsyntax-only`
- [ ] Header compiles cleanly when included in an `extern "C" {}` block with `g++`
